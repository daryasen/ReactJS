# Интернет-магазин на React

Прототип веб-приложения интернет-магазина с использованием React JS, React Router и REST API.

## Структура проекта

```
shop-app/
├── src/
│   ├── components/      # Переиспользуемые компоненты
│   ├── context/         # React Context (авторизация)
│   ├── hooks/           # Кастомные хуки (useServerGoods)
│   ├── pages/           # Страницы приложения
│   ├── services/        # Сервисы для работы с API
│   └── App.jsx          # Главный компонент с роутингом
├── server/              # Серверная часть (Node.js/Express)
│   ├── server.js        # Сервер для разработки
│   └── static.js        # Сервер для production (с раздачей статики)
└── dist/                # Собранное приложение (после build)
```

## Установка и запуск

### Разработка

1. Установите зависимости клиента:
```bash
npm install
```

2. Установите зависимости сервера:
```bash
cd server
npm install
cd ..
```

3. Запустите сервер (в отдельном терминале):
```bash
npm run server
```

4. Запустите клиент (в другом терминале):
```bash
npm run dev
```

Приложение будет доступно по адресу http://localhost:3000

### Production сборка

1. Соберите приложение:
```bash
npm run build
```

2. Запустите сервер с раздачей статических файлов:
```bash
npm run server:static
```

Или одной командой:
```bash
npm start
```

Приложение будет доступно по адресу http://localhost:3001

## Функциональность

### Страницы

- **Главная страница** (`/`) - приветственная страница с навигацией
- **Страница входа** (`/login`) - форма авторизации
- **Список товаров** (`/goods`) - каталог товаров с пагинацией (требует авторизации)
- **Описание товара** (`/goods/:id`) - детальная информация о товаре (требует авторизации)

### Авторизация

Для входа используйте:
- **Имя пользователя:** admin
- **Пароль:** admin

### API Endpoints

- `POST /api/auth/login` - авторизация
- `GET /api/goods?page=1&limit=10` - получение списка товаров с пагинацией
- `GET /api/goods/:id` - получение товара по ID

Все endpoints товаров требуют JWT токен в заголовке `Authorization: Bearer <token>`

## Технологии

- **React 19** - библиотека для создания UI
- **React Router 7** - маршрутизация
- **Vite** - сборщик и dev-сервер
- **Express** - серверная часть
- **JWT** - аутентификация

## Особенности реализации

- Кастомный хук `useServerGoods()` для работы с API
- Защищенные маршруты через компонент `ProtectedRoute`
- Обработка состояний загрузки и ошибок
- Динамическая загрузка товаров по кнопке "Загрузить больше"
- Пагинация на сервере
- Responsive дизайн
